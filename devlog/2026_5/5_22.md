Today, while switching from -bsos ns to change my bootloader from M-mode to S-mode.one to -bios default (i.e., starting to use OpenSBI for system booting), I encountered a few problems.

The first step was to change my bootloader from M-mode to S-mode. The problem was that in M-mode, I would write the HARTID, but in OpenSBI's bootloader, the startup HARTID is written to the A0 register. This caused errors when calculating the offset using PLCI. Therefore, I needed to modify the bootloader to write A0 to the TP register in start.S.

Another problem I encountered was that QEMU was set to 128MB of memory. When I was writing the `-bios none` and `-bios default` adaptations, I used different `KERNEL_BASE_LOW` values ​​for PHYSTOP calculation. However, with only 128MB of memory, using `-bios default` for PHYSTOP calculation would cause `KERNEL_BASE_LOW` to increase because I needed to reserve the lower 2MB of kernel memory for OpenSBI. This resulted in exceeding the 128MB memory limit when calculating PHYSTOP.
}
