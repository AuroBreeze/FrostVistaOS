I ran into quite a few issues today while implementing the detection for easyfs. First of all, using `virtblk_rw` requires the process context; using `rw` on its own triggers an interrupt because it needs the process ID for sleep locks and other operations, which prevents it from working.

While reviewing the xv6 source code, I noticed that xv6 initializes the file system by setting `context.ra` to the `forkret` being entered, where the file system is initialized.\

However, I feel this might pose a problem: isn’t initializing the file system for every process a bit of a performance overhead? But if we switch rw to polling and use it before the process is started, wouldn’t that be a bit clunky?

So I was thinking, why not just forge a kernel-mode process? Wouldn’t it be brilliant to allocate a `process` structure using `alloc_process`, then modify its `context.ra` to point to a function running in kernel mode, and implement the initialization of the file system within that function?Of course, you can also point directly to the fs initialization function
